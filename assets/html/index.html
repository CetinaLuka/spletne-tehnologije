Funkcionalnosti strežnika

1. Prijava

Vhod: Uporabniško ime/email, geslo

Izhod: Jwt token, podatki o uporabniku

Mobilna aplikacija bo nudila različne funkcionalnosti glede na vlogo uporabnika, zato je potrebna prijava. Pomembna je tudi zato, da so podatki varni in jih zunanje osebe ne morejo spreminjati, zato bo strežnik ob zahteval zahteval JWT token.

2. Pridobitev ponudbe

Vhod: ID ponudnika storitve

Izhod: Podatki o ponudbi

Iz baze prebere in vrne ponudbo ponudnika storitev z ustreznim ID. Aplikacija te podatke potrebuje, da lahko uporabniku pokaže kaj vse podjetje nudi. Prav tako pa tudi podjetja potrebujejo pregled nad svojo ponudbo.

3. Spreminjanje ponudbe

Vhod: ID ponudnika storitve, novi podatki o ponudbi

Izhod: Jwt token, podatki o uporabniku

Strežnik v bazu spremeni ponudbo ustreznega ponudnika.

4. Pridobitev seznama prostih/zasedenih terminov

Vhod: ID ponudnika storitve, časovno obdobje, filter (prosti/zasedeni/vse)

Izhod: Seznam prostih/zasedenih terminov

Strežnik iz baze vrne vse termine, da lahko v zasleni in uporabniki v aplikaciji vidijo kdaj so termini prosti/zasedeni.

5. Pridobitev delovnega časa

Vhod: ID ponudnika storitve

Izhod: Podatki o delovnem času

Strežnik iz podatkoven baze prebere in vrne podatke o delovnem času ustreznwega ponudnika storitve, saj bo mobilna aplikacija prikazovala delovni čas ponudnika storitve.

6. Spreminjanje delovnega časa

Vhod: ID ponudnika storitve

Izhod: Posodobljeni podatki

Strežnik v bazi spremeni delovni čas, saj lahko uslužbenci ponudnika storitve bodo spreminjajo podatke svojega podjetja.

7. Pridobitev seznama ponudnikov storitev

Vhod: opcijski filtri (lokacija, vrsta)

Izhod: Seznam ponudnikov storitev

Strežnik bo vračal seznam ponudnikov storitve, da bodo lahko stranke v aplikaciji videle vse ponudnike v bližini.

8. Dodajanje rezervacije

Vhod: ID ponudnika storitve, podatki o rezervaciji

Izhod: Dodana rezervacija

Strežnik bo v bazo dodal rezervacijo, ki jo bodo zaposleni bodo preko mobilne aplikacije vnesli.

9. Sprememba rezervacije

Vhod: ID ponudnika storitve, ID rezervacije, podatki o spremenjeni rezervaciji

Izhod: Spremenjena rezervacija

Strežnik bo v bazi posodobil rezervacijo.

10. Brisanje rezervacije

Vhod: ID ponudnika storitve, ID rezervacije

Izhod:

Strežnik iz baze izbrisal rezervacijo.

11. Pridobitev rezervacije

Vhod: ID ponudnika storitve, ID rezervacije

Izhod: Podatki o rezervaciji

Strežnik bo iz baze prebral in vrnil ustrezno rezervacijo.

12. Pridobitev informacij o podjetju

Vhod: ID ponudnika storitve

Izhod: Informacije o podjetju

Strežnik iz baze prebere in vrne podatke o ustreznemu ponudniku storitev.

13. Spreminjanje informacij o podjetju

Vhod: ID ponudnika storitve, novi podatki o podjetju

Izhod: nove nformacije o podjetju

Strežnik v bazi posodobi podatke o ustreznemu ponudniku storitev.

14. Generiranje poročila o poslovanju

Vhod: ID ponudnika storitve, časovno obdobje

Izhod: Poročilo v excel datoteki

Strežnik iz baze prebere podatke o naročilih in podjetju, ter zgenenerira poročilo o izvedenih (ali prihodnjih) naročilih za določeno časovno obdobje.

14. Pridobitev seznama strank

Vhod: ID ponudnika storitve, filtri (ime stranke, čas,...)

Izhod: Seznam strank

Strežnik iz baze prebere podatke o strankah, ki ustrezajo filtrom in jih vrne.
